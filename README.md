# 🌍 WanderLust — AI-Powered Travel & Accommodation Platform

## ✨ Features

- 🏡 Browse, create, edit and delete travel accommodation listings
- 🔐 User authentication & authorization (Passport.js)
- ⭐ Leave and manage reviews on listings
- 🗺️ Interactive maps with Mapbox
- 🖼️ Image uploads via Cloudinary
- 🤖 **AI Travel Tips** — get personalized tips for any listing
- 💬 **AI Chat Assistant** — floating chat bubble on every page, context-aware

---

## 🤖 AI Integration

WanderLust uses **[Ollama](https://hub.docker.com/r/ollama/ollama)** running locally inside Docker to power two AI features:

### 1. ✨ AI Travel Tips (`/listings/:id`)
On every listing detail page, click **"✨ Get AI Travel Tips"** to get 3 personalized travel tips generated by TinyLlama based on the listing's location, description, and price.

### 2. 💬 Floating AI Chat Bubble (all pages)
A chat assistant is available on **every page** via the red bubble in the bottom-right corner. It is:
- **Context-aware** — knows which listing you're viewing
- **Conversational** — remembers the last 6 exchanges
- **General travel assistant** — answers any travel question

### Model Used
| Model | Size | Parameters |
|---|---|---|
| [tinyllama](https://ollama.com/library/tinyllama) | 637 MB | 1.1B |

> 📦 Model is automatically pulled on first `docker-compose up`. Cached in a Docker volume so subsequent starts are instant.

### Prerequisites
- [Docker Desktop](https://www.docker.com/products/docker-desktop/) installed
- 4GB+ RAM available (for TinyLlama)
- Git

### 1. Clone the repository

```bash
git clone https://github.com/kenzo0p/Wanderlust.git
cd Wanderlust
```

### 2. Set up environment variables

```bash
cp .env.example .env
```

Edit `.env` with your real values:

```bash
# Get from https://cloudinary.com/console
CLOUD_NAME=your_cloudinary_cloud_name
CLOUD_API_KEY=your_cloudinary_api_key
CLOUD_API_SECRET=your_cloudinary_api_secret

# Get from https://account.mapbox.com/access-tokens
MAP_TOKEN=your_mapbox_token

# Any random string
SECRET=your_session_secret

# Keep as-is for Docker
ATLASDB_URL=mongodb://mongo:27017/wanderlust
OLLAMA_URL=http://ollama:11434
```

## 🧪 Running Without Docker (Local Dev)

### Prerequisites
- Node.js 22.x
- MongoDB running locally
- [Ollama installed](https://ollama.com) and running

```bash
# Install Ollama and pull the model
ollama pull tinyllama

# Install dependencies
npm install

# Set up .env (use localhost URLs)
ATLASDB_URL=mongodb://localhost:27017/wanderlust
OLLAMA_URL=http://localhost:11434

# Start the app
npm start
```

## 🔄 CI/CD Pipeline

GitHub Actions workflow (`.github/workflows/ci-cd.yml`) runs on every push to `main`:

```
Push to main
    ↓
Install & Test
    ↓
Build Docker Image
    ↓
Push to Docker Hub
    ↓
Deploy
```

To enable: add these secrets in your GitHub repo settings:
- `DOCKER_USERNAME`
- `DOCKER_PASSWORD`
